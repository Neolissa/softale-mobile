# Авторские стресс-тесты и прохождение квестов

Этот файл — ручной контур QA для авторских сценариев.  
Проверки выполняются только редакторским проходом агентом, без автоматической валидации.

## 1) Как читать стресс-тест

Для каждого квеста/курса/ивента фиксируются:
- `Angel` траектория;
- `Good` траектория;
- `Neutral` траектория;
- `Bad` траектория;
- `Harsh` траектория;
- `EarlyExit` (если применимо).

Минимум проверки:
- причинно-следственная сцепка: причина -> реакция -> последствие -> финал;
- устойчивость «голоса» персонажей;
- литературность и естественность реплик;
- отсутствие шаблонных ответов подряд;
- различие тональностей концовок (`Neutral` не равен ослабленному `Bad`, `Good` не равен ослабленному `Angel`).
- отсутствие placeholder-маркеров (`needsEditorialFill`, `временный placeholder`, `Контент шага в редактуре`);
- корректный UI-random ответов без поломки аналитики (`displayIndex -> sourceIndex`).

Статусы:
- `pass` — релизно устойчиво;
- `needs-fix` — есть конфликт логики/тона/структуры.

Блокирующие pre-release проверки:
- `npm run content:audit` обязан проходить без ошибок;
- для web-сборки обязателен `npm run web:build`;
- при провале любого из двух шагов контент в релиз не уходит.

## 6) Критерии разнообразия вариантов ответов

Для каждого узла:
- минимум 1 вариант — прямая речь;
- минимум 1 вариант — действие;
- минимум 1 вариант — пауза/молчание/отложенный формат;
- минимум 1 вариант — мета-выбор формата разговора.

Важно:
- нельзя делать все 5 опций как однотипные «заготовки-реплики»;
- реакция NPC должна отвечать на форму выбора, а не только на смысл слова;
- приоритет проверки у `Neutral` и `Good` выше, чем у крайних концовок.

## 7) Первая очередь точечной переписи (первичный fix-queue)

`needs-fix` в первую очередь:
- `cinderella-advocate`: уплотнить литературность в сжатых узлах этапов 3-5.
- `first-word-forest`: усилить индивидуальные голоса оппонентов в этапах 2-4.
- `partisan-hq`: добавить более выраженные последствия для `Neutral`.
- `dragon-ultimatum`: развести тон `Good` и `Neutral` в середине арки.
- `stop-crane-train-18plus`: усилить эмоциональную цену выбора в этапе вагонетки без натуралистичности.

Порог перехода узла в `pass`:
- устранена шаблонность;
- реакция NPC меняет сцену, а не повторяет мораль;
- финальная ветка логично вытекает из накопленных решений.
